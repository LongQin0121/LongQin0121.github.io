What is a Hyperlink ?

Hyperlinks represent the cornerstone of the World Wide Web, offering a means for users to navigate between pages, documents, or even within the same document. They are embedded within web pages as clickable elements, which direct the user to a different location or action when selected. This navigational mechanism not only shapes the structure of the web but also defines its interconnected nature.[1]

Click Hyperlink

 

How to use it?

 

A hyperlink can be added to a webpage by wrapping the text in a the following HTML code:


                <a href=”referencefilepath”>ANCHOR TEXT</a>

The hyperlink code includes the following components:

  • An opening anchor tag ( <a> )
  • The type of anchor is then defined by “href” which stands for “Hypertext Reference”
  • The referenced URL or file location is then included in the quotes. This will commonly be the web address of the page you want to link to
  • The anchor text is located between the opening and closing “a” tags and defines the linked text that will display on the webpage.
  • The opening anchor tag is then closed

Http Request

 

The Role of HTTP Requests

Hyperlinks rely on the Hypertext Transfer Protocol (HTTP) to request and receive resources across the web. When a hyperlink is activated, the browser sends an HTTP request to the server hosting the target resource. The server then responds with the content, which the browser renders for the user.

 

Http Request - Examples

 

HTTP Method Idempotent Safe Has Body Typical Use Case
GET Yes Yes No Retrieve data not changing the server state.
POST No No Yes Submit data to the server (e.g., forms).
PUT Yes No Yes Replace a resource or create it if it doesn't exist.
DELETE Yes No No Remove a specified resource.
PATCH No No Yes Partially update a resource.

Ajax

 

Advanced Hyperlink Usage

Beyond basic navigation, hyperlinks can trigger more complex interactions using JavaScript and AJAX (Asynchronous JavaScript and XML). This allows for dynamic content loading without the need to reload the entire page, enhancing the user experience by providing instant feedback and interaction.[7]

 

As you can see from the above image,the Ajax includes the following steps:

  1. Event: An event occurs in a web page (e.g., the page is loaded, or a button is clicked).
  2. XMLHttpRequest Object: JavaScript creates an instance of the XMLHttpRequest object. This object allows the sending of HTTP requests to the server in the background.
  3. Server Interaction: JavaScript uses the XMLHttpRequest object to send a request to a web server. The server processes the request, often querying data from databases or performing operations.
  4. Response: The server sends a response back to the web page. The response can be in various formats, including XML, JSON, HTML, or plain text.
  5. JavaScript Callback: Upon receiving the server response, a JavaScript function (often called a callback) is called to handle the response.
  6. Update Web Page: The callback function processes the server response and can then update the content of the web page dynamically by modifying the DOM (Document Object Model) without needing to reload the page.

Ajax Example - Public API

 

Public API Ajax VS local Ajax

  1. A Public API AJAX involves making HTTP requests over the internet to a third-party API server to fetch data. It's useful for accessing frequently updated data sources like weather, news feeds, or social media, but requires adhering to the API's rules, authentication, and rate limiting.
  2. Local JSON AJAX involves fetching data from a local JSON file within your application's file structure, without making requests over the internet. It's useful for static data, mocking API responses during development, and scenarios where you have full control over the data updates.
  3. The main differences lie in the data source (remote vs. local), data ownership and control, data freshness, authentication requirements, and scalability/performance considerations.[7]
